# Yaxiio 质量宪章

> 版本: 2.0 | 日期: 2026-05-30
> 适用范围: Yaxiio 自身开发 + 全部 Agent 产出
> 基于 Martin Fowler、Google Readability、Nielsen 可用性原则、工业检测标准

---

## 零、为什么需要质量宪章

Yaxiio 是一个会自我进化的 Agent 系统。如果它产出的东西没有质量门禁，进化方向就是随机的。

但"质量"对不同产物意味着完全不同的东西：

| 产物类型 | 质量意味着 | 反面 |
|---------|----------|------|
| **代码** | 可维护、可读、可扩展 | 圈复杂度爆炸、零注释 |
| **图片** | 分辨率达标、无水印杂色、主体清晰 | 模糊、变形、杂色水印 |
| **文章/翻译** | 术语一致、无遗漏、语法正确 | 漏译、混杂语言、术语混乱 |
| **文档/方案** | 结构完整、逻辑清晰、可执行 | 只有空壳、关键章节缺失 |
| **数据/配置** | Schema 正确、字段完整、无冗余 | 缺必需字段、格式错误 |
| **视频** | 分辨率达标、音画同步、无黑帧 | 卡顿、不同步、黑屏帧 |

**一套统一的评估框架，按产物类型分维度。**

## 一、执行机制

```
任何 Agent 产出的内容
       │
       ▼
┌──────────────────┐
│ UnifiedScorer    │  ← 自动识别产物类型
│ 根据类型选择维度  │
└──────┬───────────┘
       │
       ▼
  score ≥ 6 → 交付 / 合入
  score < 6 → 打回 + 差距分析 + 自动修复
  score < 4 → 暂停 + 通知人工审核
```

**统一规则**：

| 规则 | 适用对象 | 说明 |
|------|---------|------|
| 提交前自检 | 人类开发者 | `python3 tools/qa_check.py` 在 commit 前运行 |
| CI 门禁 | 所有 PR | 低于 6 分阻止合并 |
| AI 产出门禁 | 所有 Agent | 低于 6 分自动打回修复 |
| 人工兜底 | score < 4 | 暂停自动流程，通知人工 |
| 宪法级强制 | 全部 | 质量规则不可绕过 |

## 二、按产物类型的质量维度

### 2.1 代码 (Code)

产物: `.py`, `.ts`, `.vue`, `.js`, `.yaml`, `.json`

| 维度 | 权重 | 检测方法 | 硬阈值 |
|------|------|---------|--------|
| 可维护性 | 20% | 圈复杂度 + 耦合度 + 文件大小 | CC ≤ 10, ≤ 500行 |
| 可读性 | 20% | 命名规范 + 注释率 + 函数长度 | 注释率 10-30%, 函数 ≤ 40行 |
| 可扩展性 | 15% | 开闭原则 + 接口抽象度 | 硬编码 if-else ≤ 3层 |
| 简洁性 | 15% | 冗余代码率 + 过度设计检测 | 冗余 ≤ 10% |
| 灵活性 | 10% | 配置化程度 + 多态使用 | 硬编码率 ≤ 20% |
| 可复用性 | 10% | DRY 违规率 + 模块复用 | DRY 违规 ≤ 5% |
| 可测试性 | 10% | 依赖注入率 + 纯函数比例 | 注入率 ≥ 80% |

### 2.2 图片 (Image)

产物: `.png`, `.jpg`, `.webp`, `.svg`

| 维度 | 权重 | 检测方法 | 硬阈值 |
|------|------|---------|--------|
| 技术质量 | 25% | 分辨率、压缩率、格式匹配 | 宽度 ≥ 800px, JPEG quality ≥ 80 |
| 内容完整性 | 25% | 主体检测、边缘裁剪、变形 | 主体覆盖率 ≥ 60%, 无变形 |
| 洁净度 | 20% | 水印检测、杂色噪点、伪影 | 无水印, 噪点率 ≤ 2% |
| 品牌一致性 | 15% | 色板匹配、Logo 位置、风格统一 | 主色偏差 ≤ ΔE 10 |
| 元数据 | 15% | alt文本、尺寸标注、来源 | alt 非空, 有尺寸 |

### 2.3 文章/翻译 (Text)

产物: 翻译稿、产品描述、博客文章、SEO 内容

| 维度 | 权重 | 检测方法 | 硬阈值 |
|------|------|---------|--------|
| 准确性 | 30% | 术语一致性、事实正确性 | 术语一致率 ≥ 95% |
| 完整性 | 25% | 原文覆盖率、无遗漏段落 | 覆盖率 ≥ 98% |
| 语言质量 | 20% | 语法检查、流畅度、自然度 | 语法错误 ≤ 2 处/千字 |
| 格式规范 | 15% | Markdown/HTML 结构、链接有效性 | 无断链, 层级正确 |
| SEO 要素 | 10% | 标题层级、meta描述、关键词密度 | h1 唯一, meta 非空 |

**术语一致性自动检测**：
```
原文: "EPDM rubber gasket" → 参考术语库 → 期望: "EPDM橡胶密封垫片"
检测: 翻译中是否出现了 "EPDM橡胶垫圈" 或 "三元乙丙垫片" → 不一致，标记
```

### 2.4 文档/方案 (Document)

产物: 设计文档、实施计划、分析报告、审核清单

| 维度 | 权重 | 检测方法 | 硬阈值 |
|------|------|---------|--------|
| 结构完整性 | 30% | 必需章节检查、目录层级 | 必需章节全部存在 |
| 内容深度 | 25% | 每章节字数、关键要素覆盖 | 每章 ≥ 50字, 有关键要素 |
| 可执行性 | 20% | 是否有具体步骤/命令/时间 | 至少 3 条可执行项 |
| 逻辑一致性 | 15% | 前后矛盾检测、数字一致性 | 无处矛盾 |
| 格式规范 | 10% | Markdown 规范、表格完整、链接有效 | 表格完整, 链接有效 |

**必需章节检查**：

| 文档类型 | 必需章节 |
|---------|---------|
| 设计方案 | 背景 → 目标 → 架构 → 实现 → 风险 → 路线图 |
| 实施计划 | 目标 → 步骤 → 时间线 → 责任人 → 验收标准 |
| 分析报告 | 数据 → 方法 → 发现 → 结论 → 建议 |
| 审核清单 | 项目列表 → 每项有状态标记 → 总计/通过率 |

### 2.5 数据/配置 (Data)

产物: JSON、YAML、能力卡片、i18n 文件、数据库记录

| 维度 | 权重 | 检测方法 | 硬阈值 |
|------|------|---------|--------|
| Schema 合规 | 30% | 字段类型、必需字段、枚举值 | 必需字段 100% 存在 |
| 数据完整性 | 25% | 空值率、字段数量一致性 | 空值率 ≤ 5% |
| 语义正确性 | 20% | 值域范围、逻辑关系 | 无越界值 |
| 无冗余 | 15% | 重复记录、无用字段 | 重复率 ≤ 1% |
| 可追溯性 | 10% | 时间戳、作者、来源 | 有 timestamp |

**Yaxiio 自身案例**——页面 i18n 数据检查：
```
✅ 检查: solar-farm-ground-screw.json
   - Schema 合规: 18/18 必需字段存在
   - 数据完整性: 0 空值
   - 同层级一致性: 18 keys (与同级 6 个文件的 key 数量一致) ✅
   
🔴 检查: 螺旋地桩.json（中文文件名）
   - 语义正确性: 文件名应为英文 slug "solar-farm-ground-screw"
   - 元数据缺失: 无 updated_at 字段
```

## 四、开发工作流集成

```bash
# 每次 commit 前运行
python3 tools/qa_check.py --changed        # 只检查变更
python3 tools/qa_check.py --all             # 全量检查
python3 tools/qa_check.py --type image      # 只检查图片

# 输出示例:
# ✅ constitution.py: 7.6 — maintainability=8, readability=8, ...
# 🔴 workflow_engine.py: 4.0 — 3 violations:
#    - CC=12 (函数 _orchestrate_subtasks, 阈值=10)
#    - 文件 1231 行 (阈值=500)
#    - 嵌套 5 层 (阈值=4)
# ✅ hero-power.png: 8.2 — resolution=1200x800, no watermark, brand ok
# 🔴 螺旋地桩.json: 5.5 — filename not English slug
```

## 五、一句话

> 代码的质量标准是圈复杂度和可读性。图片的质量标准是分辨率和洁净度。翻译的质量标准是术语一致和完整性。不同的东西，不同的尺子——但过的是同一道门。Yaxiio 自己先过这门，才有资格衡量 AI 的产出。
