## 软件的主要功能

1. **多智能体并行调度**
   Commander 总指挥接收任务后，通过宪法审查机制对系统管理操作开放白名单直通，对其余业务操作强制路由至五层流水线。L1 感知层结合关键词匹配与大语言模型识别任务意图并计算置信度；L2 规划层将任务拆解为子任务依赖图，无依赖关系的子任务由线程池并行发射到对应专业 Agent；L3 调度层按需启动 Agent 神经元进程并分发任务到 Redis 频道；L4 执行层在沙箱环境中运行并轮询等待 Agent 回复，超时自动降级；L5 评估层对执行结果进行规则基础分与大语言模型深度评分的加权计算，低分任务自动标记待进化。全流程状态实时写入 Redis 状态机，包含五层里程碑、子任务执行耗时和产出哈希，支持 Commander 进程重启后断点扫描与自动恢复。

2. **多语翻译引擎**
   翻译 Agent 加载术语词典作为系统提示词，支持中文到英语、俄语、阿拉伯语、西班牙语、法语共五种目标语言的一致翻译，强制遵循术语词典确保全站术语统一。默认使用低推理深度模式追求响应速度，支持调用方通过任务参数覆盖模型选择和推理深度。翻译结果积累至 Agent 记忆形成持续优化的知识库。

3. **内容质量审计**
   审计 Agent 对网站源代码进行全量扫描，自动检查设计合规性、术语一致性、旧版 UI 残留。审计结果按文件维度汇总，标记每个文件的代码行数、设计系统命中数、旧版 UI 残留数和主题覆盖度，结合大语言模型生成可读的分析报告和修复建议，报告自动保存至黑板报告库并支持历史对比追踪改进幅度。

4. **产品搜索与报价生成**
   售前 Agent 按行业赛道、产品品类、规格参数等维度检索产品数据库，支持跨品类对比。通过大语言模型理解自然语言查询意图，匹配数据库记录，返回结构化的产品规格、材质、表面处理和参考价格区间。结合客户需求（数量、目的地、交期）可生成报价草稿。

5. **UI/UX 设计生成**
   内置 UI/UX 设计师、品牌策略师、前端工程师三个专业智能体。设计师基于品牌设计令牌（包含品牌色、字体、间距、阴影等）自动生成网页布局方案和组件样式建议；策略师分析品牌调性、竞品设计模式和配色升级方案；前端工程师评估设计方案的技术可行性与响应式适配方案。完整重设计任务自动拆解为启发式评估、竞品分析、品牌调性转译、首页布局重构、产品页重设计、移动端适配建议、设计规范输出共七个子任务，按依赖关系并行编排执行。

6. **自我监控与故障修复**
   Guardian 守护进程每三十秒对 Commander 进行健康检查并定期评估调度质量，低分触发渐进式修复。Agent 崩溃时 Commander 自动重启进程，连续低分时派出系统医生 Agent。系统医生加载诊断技能，读取故障 Agent 的记忆和提示词文件，通过大语言模型分析根因并生成多个修复候选方案，A/B 对比测试后自动选出最优方案更新配置。双 Guardian 通过 Redis 分布式锁实现主备选举，主节点故障时三十秒内自动接管。

7. **任务状态追踪**
   每个任务完整生命周期记录于 Redis 状态机，实时可查当前流水线层级、五层各自完成状态、子任务执行 Agent 和耗时、不可变追加的完整事件时间线。大体积子任务产出独立存储避免主状态膨胀。Dashboard 提供可视化监控界面，Commander 重启后自动扫描未完成任务并重新调度。

## 软件的技术特点

1. **五层 MCP 流水线架构** — 感知层、规划层、调度层、执行层、评估层各司其职，层间通过标准 JSON-RPC 协议通信，可独立升级替换

2. **神经元式 Agent 运行时** — 每个 Agent 是独立进程，拥有感知器(Redis订阅)、大脑(大语言模型)、双手(工具调用)、嘴巴(结果发布)、记忆(Redis存储)的完整回路

3. **任务状态机与事件溯源** — 每个任务的五层里程碑、子任务进度、执行耗时实时记录，时间线不可变追加，支持断点恢复

4. **并行依赖编排** — 复杂任务自动拆解为子任务 DAG，无依赖的子任务并行发射到线程池，最大化执行效率

5. **宪法模式** — Commander 受白名单约束，业务操作强制走五层流水线，禁止越权直接执行，确保架构一致性

6. **双层互保** — 双 Guardian 进程通过 Redis 分布式锁实现 leader 选举，主备自动切换，消除单点故障

7. **模型路由** — Agent × 任务类型 × 推理深度三级映射，翻译用低推理模式追求速度，审计用高推理模式确保质量

8. **自我进化闭环** — Agent 质量低→系统医生诊断→分析提示词→生成候选→A/B测试→自动选优，无需人工介入
