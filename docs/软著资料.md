# 雅溪 Yaxiio 智能调度系统

> 版本 v1.04 | AGPLv3 | 2026-05

## 一、软件概述

**软件名称**: 雅溪智能调度系统（Yaxiio Intelligent Orchestration System）

**简称**: Yaxiio

**开发语言**: Python 3.12 | TypeScript

**代码规模**: 21 核心模块，约 15,000 行

**运行环境**: Docker + Ubuntu 24.04 + Redis + SQLite

## 二、技术架构

### 五层模块化架构

```
┌─────────────────────────────────────────┐
│ L5 进化层: GEPA提示词优化、A/B测试、DSPy互补 │
├─────────────────────────────────────────┤
│ L4 评估层: LLM-as-Judge、全链路追踪、Prometheus│
├─────────────────────────────────────────┤
│ L3 工作流层: 任务状态机、并行调度、断点续传、条件分支│
├─────────────────────────────────────────┤
│ L2 智能体层: Agent工厂、生命周期、RAG、分层记忆、多Provider│
├─────────────────────────────────────────┤
│ L1 基础组件层: Redis、SQLite、MCP、Skill热加载、向量DB│
└─────────────────────────────────────────┘
```

### 创新点

1. **五层模块化架构**: 层间通过标准接口通信，独立升级，互不干扰
2. **LLM驱动的自进化**: 系统可读取自身代码，通过LLM生成补丁，语法验证后自动应用
3. **沙箱演习机制**: 低分任务在隔离环境中模拟优化，对比指标后自动应用改进
4. **GEPA算法**: Generate→Evaluate→Pick→Apply 全生命周期Prompt优化
5. **Agent角色继承**: 新Agent自动继承父角色能力链，FullStack可继承8项能力
6. **任务状态机**: 11状态流转，Redis持久化，Commander重启后自动恢复
7. **全链路追踪**: 每次LLM调用的Token、延迟、Agent全部记录

## 三、核心功能

| 功能 | 说明 |
|------|------|
| 多Agent协作 | Redis PubSub + A2A协议，支持10+ Agent并行 |
| 任务拆解 | LLM驱动的模糊意图→原子任务序列 |
| 自主审计 | 扫描代码→LLM分析→生成结构化报告 |
| 代码修复 | 读取诊断报告→LLM生成方案→沙箱执行 |
| 自进化 | 读自身代码→LLM发现缺口→生成补丁→验证应用 |
| 沙箱演习 | 隔离环境中重放低分任务，优化流程 |
| 外部调用 | MCP JSON-RPC 2.0，Dashboard API，WebSocket |
| 两层守护 | PM2→Guard→Commander，防崩溃+自动恢复 |
