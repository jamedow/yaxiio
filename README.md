# 雅溪 Yaxiio — Agent 操作系统内核

> **你的 Agent 团队每天都在变聪明。你不需要在场。**

Yaxiio 不是又一个 Agent 框架。它是一个**会自我进化的 Agent 操作系统**——宪法约束管行为边界，模板克隆保上下文隔离，仓颉记忆喂进化引擎。用的人越多，系统越聪明。

---

## 为什么是 Yaxiio？

其他 Agent 框架解决"怎么让 LLM 调用工具"。Yaxiio 解决"**一群 Agent 怎么像一个团队一样自我管理、自我进化，而不需要人类在中间协调**"。

这不是"Agent 协作"，这是 **Agent 自治**。

## 一秒看懂架构

```
用户请求 → Gateway ──→ Commander (宪法审查)
                         │
         ┌───────────────┼───────────────┐
         ▼               ▼               ▼
     L1 感知         L2 规划          L3 调度
   "要干什么?"     "怎么拆?"       "谁来做?"
         │               │               │
         └───────────────┼───────────────┘
                         ▼
                     L4 执行 ──→ Agent 池 (独立进程, 沙箱隔离)
                         │
                         ▼
                     L5 进化 ──→ 评分 → 差距分析 → 经验回写
                                   │
                         ┌─────────┴─────────┐
                         ▼                   ▼
                    仓颉记忆              DSPy 优化
                 (三层记忆架构)        (自动 Prompt 编译)
```

**每一层都有独立的 TraceLogger, 同一个 trace_id 贯穿全链路。**

## 五大护城河

### ⚖️ 宪法约束
硬编码在 Commander 入口。任何 Agent 的任何操作，先过宪法。四种判决：ALLOWED / DELEGATED / REJECTED / DEGRADED。违宪行为记录到 Redis，审计不可绕过。**业界唯一在运行时层做硬约束的系统。**

### 🧬 模板克隆 + 仓颉记忆 + 自进化
每任务创建隔离 Agent 实例 → 用完销毁，上下文零泄露 → 成功经验提取到 L0 经验库 → Chroma 语义索引 → 下次同类任务自动注入 → DSPy 持续优化 Prompt → 高分模板回写。**这是一条闭合的数据飞轮。用 6 个月的 Yaxiio 和刚装的 Yaxiio，质量天壤之别。**

### 🎴 能力卡片 + 结构化学符通信
Agent 的"身份证 + 简历 + 合同"三合一 JSON。卡片定义 input/output Schema → Agent 间用结构化五元组 `<task_id, agent_type, schema, depends>` 通信 → 不靠 LLM 翻译 → Token 消耗降 80%。卡片可导出、可跨组织复用。

### 🛡️ 防呆设计 — 业界独有

Yaxiio 是唯一将防呆设计（Poka-Yoke）作为系统内核哲学的 Agent 框架。从平民到专家，同一套系统自适应复杂度：

| 等级 | 看到什么 | 体验 |
|------|---------|------|
| 🟢 平民级 | 4 个字段 | "翻译这个到阿拉伯语" → 系统自动搞定一切 |
| 🔵 维护级 | 10 个字段 | 能调 quality、model、max_retries |
| 🟣 技术级 | 20 个字段 | 精确控制 temperature、few_shot、评分权重 |
| ⚪ 大神级 | 全部字段 | 改宪法、调协议、联邦部署 |

**核心理念**：不是"你不能调"，而是"你不用调，我已经帮你调好了"。

### 📊 五层 MCP + 全链路日志
每一层独立进程，独立端口。同一个 trace_id 贯穿 L1→L5。`/trace/:id` 拉出完整决策链路："L1 为什么判定为 audit？L2 为什么选了审计官？L5 为什么给了 7.5 分？"——学者做论文、IoT 工程师调产线，不靠猜。

## 谁用了会变好

不是"谁在用"，是"谁用了之后，原来的痛点会消失"。

| 你正在用的工具 | 现在的痛点 | 接入 Yaxiio 之后 |
|--------------|-----------|-----------------|
| 🧠 **Trae / Cursor / Copilot** | 任务拆成 5 步，第 3 步卡住 → 剩下全卡住。跑完不知道实现了多少需求。 | L3 异步调度：一步卡住不阻塞其他。L5 自动评分 + 差距分析：跑完立刻告诉你"实现了 80%，还差 X 和 Y"。 |
| 🔗 **LangChain / CrewAI** | Agent 记忆跨任务累积膨胀，越用越慢、越用越乱。 | Template Clone：每任务新建 Agent 实例，用完销毁。经验提取到 L0 向量库，下次只注入精华。 |
| 🤖 **AutoGPT / Claude Code** | 自主循环但无质量门禁。跑 10 轮不知道哪轮好、哪轮在兜圈。 | L5 统一评分总线：每轮评分 → 低于 7 分触发差距分析 → 自动换策略重试。3 轮无改善自动停。 |
| 🏗️ **Jenkins / Airflow** | 流水线是人预先画的 DAG。新任务类型来了 → 报错"找不到流程"。 | Commander 自己设计 DAG：LLM 拆解任务 → L3 按依赖并行调度 → L5 发现瓶颈自动重组。 |
| 🌐 **外贸 B2B** | 询盘 → 人工匹配产品 → 人工报价 → 人工翻译。一条询盘半天。 | 询盘 → L1 意图理解 → L2+L3 产品匹配 → L4 技术方案 + 翻译 → L5 评分 → 人只确认最后一步。 |
| 🏭 **IoT / 树莓派** | 传感器异常靠人看。出问题追溯靠猜。 | SQLite 版 200MB 跑在树莓派上。L1 感知异常 → L2 诊断 → L3 调度修复 → L4 执行 → L5 评估。全程 trace_id 可追溯。 |

## 未来展望

> Yaxiio 不赌 Agent 会变得更"强大"，赌 Agent 会变得"更聪明"。
> 不是靠人类写更多规则，是靠系统自己从每一次任务中学习。

| 时间 | 里程碑 |
|------|--------|
| 1 年内 | 100+ 生产实例验证数据飞轮，经验库破万条，Chroma 检索命中率 60%+ |
| 2-3 年 | 能力卡片市场自运转，跨行业、跨组织交易，L5 进化沉淀金牌模板 |
| 5 年+ | Agent 世界的 Linux：Desktop / Cluster / Lite / Lab |

Agent 领域未来分两条路：**功能堆叠型**（功能可复制，护城河是沙子）vs **自我进化型**（数据飞轮越转越快，护城河是石头）。

Yaxiio 的路是第二条。
